# HariWorks - Project Constitution

## Vision

HariWorks is a premium personal brand website built for Hariprasad.

The website should communicate:

> This developer builds real-world software solutions.

The website is NOT a portfolio template.

It is a production-quality personal website.

---

## Core Values

- Minimal
- Professional
- Elegant
- Modern
- Fast
- Accessible
- Responsive
- SEO Friendly
- Maintainable

---

## Development Rules

- Use Server Components by default.
- Use Client Components only when necessary.
- Use strict TypeScript.
- Never use `any`.
- Never duplicate components.
- Every component must be reusable.
- Keep business logic separate from UI.
- Follow Next.js App Router best practices.
- Follow accessibility best practices.
- Keep Lighthouse above 95.

---

## Folder Structure

src/
app/
components/
content/
hooks/
lib/

utils/

---

## Component Rules

Every component should:

- have one responsibility
- be reusable
- be typed
- be documented
- support dark mode

---

## Design Rules

- Apple inspired whitespace
- Linear inspired typography
- Vercel inspired developer experience

Do not copy any website.

Create a unique design.

---

## Performance Rules

- Lazy load where appropriate
- Optimize images
- Dynamic imports when useful
- Minimize client-side JavaScript

---

## Accessibility

- Keyboard navigation
- Proper heading hierarchy
- ARIA labels
- Focus states
- Color contrast
- Reduced motion support
